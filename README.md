**이름:** [선재민]

> 본 프로젝트의 목표는 제 전공(기계공학) 및 동아리(EV 포뮬러 자작차) 활동에서 얻은 아이디어를 바탕으로, Python과 ROS를 활용하여 차량의 조향 기하학을 시뮬레이션하는 프로그램을 만들어 보기입니다.

 제가 직접 사용할 프로그램이며 다음년도 대회에 참여하게 될 후배들이 예상 사용자입니다.

---

## 1. 프로젝트 개요
### 1.1 문제 정의
- 조향 설계 과정에서 **아커만 조향 기하학(Ackermann Steering Geometry)**은 선회 시 타이어 마모를 줄이고 안정적인 코너링을 결정하는 핵심 요소입니다.
- 하지만 수식으로만 계산된 설계값이 실제 주행 시 바퀴의 궤적이나 회전 중심점(ICR)에 어떤 영향을 미치는지 직관적으로 확인하기 어렵습니다.
- 또한 실제 차량 제작 과정에서 프레임을 설계했던 치수대로 정확하게 제작하기 어려워 그에 맞는 설계값을 다시 계산해야 할 경우가 많습니다.

### 1.2 필요성 및 목표
- **필요성:** 현재 제작 중인 EV 포뮬러 차량(휠베이스 1500mm, 폭 420mm)의 조향 시스템 설계 검증이 필요합니다.
- **목표:** 설계 단계에서 조향 암(Steering Arm)의 위치 변경에 따른 아커만율 변화를 시뮬레이션하여 최적의 설계점을 도출하고자 합니다.

---

## 2. 핵심 기능
- **실시간 조향 연산:** 핸들 각도 입력 시 아커만 공식을 적용하여 좌/우 앞바퀴의 개별 조향각을 실시간으로 계산해줍니다.
- **차량 제원 설정:** 휠베이스(L) 및 트레드(W) 값을 변수로 설정하여 다양한 차량 모델에 대응 가능하게 하려합니다.
- **ROS 시각화 연동:** 계산된 데이터를 JointState 메시지로 발행하여 ROS2 RViz 환경에서 3D 차량 모델의 바퀴 움직임을 시각화할 예정입니다.
- **궤적 예측:** 조향각을 바탕으로 예상 회전 반경과 주행 경로를 화면에 표시합니다.

## 3. 결과물 형태
- 프로젝트의 필요성과 목표를 고려해 보았을 때 CLI 형태의 결과물도 충분하다고 생각하여 결과물의 형태는 CLI로 결정하였습니다. 추후 프로젝트를 발전시킬 경우 사용자 편의를 위해 결과물 형태를 GUI로 변경하는 것을 고려하고 있습니다.

## 4. 사용할 Python 개념
- 입출력 및 형변환 (input(), float()):
사용자로부터 차량 제원(휠베이스, 폭)과 조향각을 입력받고 연산을 위해 실수형으로 변환.

- 수학 모듈 (math):
math.radians()로 각도를 변환하고 math.tan(), math.atan()을 사용하여 아커만 공식 구현.

- 제어문 (if, while):
while 루프를 사용해 사용자가 '종료'를 입력하기 전까지 반복해서 시뮬레이션 수행.
if문을 통해 물리적 조향 한계를 벗어나는 입력값에 대한 예외 처리.

- 데이터 구조 (list, dictionary):
입력한 각도에 따른 안쪽/바깥쪽 바퀴의 결과값을 리스트에 누적 저장하여 시뮬레이션 이력 관리.

- 함수 (def):
calculate_ackermann(angle) 처럼 계산 로직을 함수로 분리하여 코드의 재사용성과 가독성 확보.
